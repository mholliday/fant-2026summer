1. Who this guide is for

This guide covers the administrator-only features of FANT. Everything a regular user can do is covered in the User Guide — this document only adds the extra powers that come with an admin account: managing users and their permissions, and the full lifecycle of donor records (including restoring and permanently deleting them) plus database backups.

2. Reaching the Admin Panel

When you're logged in with an admin account, an Admin Panel link appears in the top navigation bar. It lists every user account and holds the database backup/restore controls. If you don't see the link, your account doesn't have admin access.

3. Access levels & permissions explained

Each user has an access level: a number from 0 to 7. Under the hood it's a set of bit flags, where each bit switches one capability on or off. The Admin Panel shows both the 4-digit binary form (e.g. 0110) and the number.

BitCapabilityValueMeaning
bit 2Read4View donor records, search, export PDFs
bit 1Write2Create, edit, archive donors; manage images
bit 0Admin1Admin Panel: users, backups, restore/delete archived records
bit 3Immutable8Protected account — cannot be edited or deleted (see §6)

You add the values together to combine capabilities. The most useful combinations:

LevelBinaryGrantsTypical role
00000NothingDisabled / no access
40100Read onlyViewer / read-only researcher
60110Read + WriteLab staff entering and editing records
70111Read + Write + AdminAdministrator
Because capabilities stack, an admin who should also enter data needs 7 (read + write + admin), not 1. A level of 1 grants the Admin Panel but not the ability to read or edit donor records.

The New User form only accepts levels 0–7, so the immutable bit (8) can't be assigned through the interface.

4. Creating a user

  1. Open the Admin Panel.
  2. Click + New User at the top right.
  3. Fill in First Name, Last Name, Username, and a Password.
  4. Set the Access Level (0–7) using the table in §3 — for example 6 for a data-entry user or 7 for another administrator.
  5. Click Create User. The new account appears immediately in the list.
Give the user their username and initial password directly. They can be prompted to change it, and you can reset it at any time (§5).

5. Resetting passwords & deleting users

In the Admin Panel each user row has an Actions column:

  • Reset Password — opens a form to set a new password for that user.
  • Delete — permanently removes the account after a confirmation prompt. This cannot be undone.
Deleting a user removes their login but does not delete the donor records they created — those remain, still attributed to them.

6. The protected super-admin

An account whose access level has the immutable bit set (value 8, e.g. level 15) is a protected super-admin. Its row shows "Protected super-admin" instead of action buttons — it can't be edited or deleted through the interface. This guarantees there is always at least one working admin account. These accounts are provisioned outside the app, not through the New User form.

7. Creating and editing donors

Admins create and edit donors exactly as write-access users do — see the User Guide for the full walkthrough. In short: use + New Donor on the Dashboard to create one, or Edit on a donor page to change it. Both the Skeletal Inventory and Skeletal Analysis sections, images, dentition, and osteometry are entered through the same form.

8. Archiving, restoring & deleting donors

Archive

Any write-access user can archive a donor: open it and click Archive. The record leaves the main Dashboard but is not deleted — it moves to the archive (View Archive on the Dashboard).

Restore (admin only)

Open an archived donor and click Restore to return it to the active Dashboard.

Delete permanently (admin only)

Open an archived donor and click Delete Permanently. After you confirm, the record and its history are removed for good.

Permanent deletion cannot be undone. Archive rather than delete unless you are certain the record should be gone. Consider taking a backup (§10) first.

9. Version history & rollback

Every save creates a version, listed under Version History at the bottom of a donor page. As an admin you'll see a Restore button next to older versions. Clicking it rolls the record back to that version after a confirmation.

Restoring an older version deletes the versions that came after it. Newer edits are discarded, so be sure before you roll back.

10. Backing up and restoring the database

The Admin Panel has two database controls above the user list:

  • Backup Database — downloads a single .json file containing all donors, versions, and users. Store it somewhere safe. Do this regularly and before any risky operation.
  • Restore from Backup — choose a previously downloaded backup file to load.
Restore replaces everything. Loading a backup overwrites all current donors, versions, and users with the contents of the file. You'll be asked to confirm first. When it finishes, FANT reports how many donors, versions, and users were restored.
